## The Risk

`docker compose restart` just stops and restarts the container — it doesn't wipe volumes. So the keys themselves (stored in the database) **will survive**, as long as your PostgreSQL data is in a named volume or bind mount, which it almost certainly is in any standard Canvas Docker setup.

The keys generated by `Lti::KeyStorage.rotate_keys!` are stored in the **database**, not on the container filesystem, so they persist across restarts.

---

## What Would Actually Lose the Keys

You'd only lose the rotated keys if you ran something destructive like:

```bash
docker compose down -v   # the -v flag destroys volumes
docker compose rm -v
# or manually dropped and recreated the DB
```

A plain restart, stop/start, or even recreating the container without `-v` is safe.

---

## Verify Before Restarting

If you want to be sure, you can confirm the keys are in the DB before and after:

```bash
# Before restart
docker exec -it <canvas-container> bundle exec rails runner \
  "puts Lti::KeyStorage.present_key.public_key.n.num_bits"

# After restart, run the same command and compare
```

If the bit count matches, your rotated key survived the restart.
